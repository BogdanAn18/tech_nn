# 💸 Монитор подписок

**Фулстэк‑приложение для управления личными подписками**  
*Отборочное задание направления «Фулстэк»*

Проект представляет собой платформу для централизованного контроля регулярных расходов. Он состоит из трёх взаимодействующих компонентов:

- **Серверная часть (Backend)** — построена на Node.js (Express) с использованием PostgreSQL и JWT-аутентификации.
- **Веб-интерфейс (Frontend)** — разработан на Next.js с графиками Recharts.
- **Мобильное приложение** — создано на React Native (Expo) с нативной навигацией.

Все части синхронизируются через единое REST API: изменения, внесённые в вебе или мобильном приложении, немедленно отражаются на всех клиентах.

---

## 🧠 Архитектура приложения

![Схема взаимодействия](https://via.placeholder.com/600x200?text=Backend+<->+Frontend+<->+Mobile+API)

1. **Клиентские приложения** (фронтенд и мобильная версия) отправляют HTTP-запросы к бэкенду.
2. **Сервер** обрабатывает запросы, проверяет авторизацию (JWT) и выполняет операции с базой данных.
3. **PostgreSQL** хранит информацию о пользователях и их подписках.
4. Благодаря единому API достигается полная синхронизация данных между платформами.

## 🗄️ Установка и настройка PostgreSQL

### Загрузка и установка

1. Перейдите на [официальный сайт PostgreSQL](https://www.postgresql.org/download/windows/) и выберите установщик для вашей операционной системы.
2. Запустите скачанный файл и следуйте инструкциям.
   - **Пароль суперпользователя**: обязательно запомните или запишите его — он потребуется для подключения к базе.
   - **Порт**: оставьте значение по умолчанию `5432`.
   - Остальные параметры можно оставить без изменений.
3. После завершения установки снимите галочку «Launch Stack Builder» и нажмите **Finish**.

### Создание базы данных

1. Откройте **pgAdmin 4** (графический клиент PostgreSQL).
2. Введите пароль суперпользователя, заданный при установке.
3. В левой панели раскройте **Servers → PostgreSQL 16**.
4. Щёлкните правой кнопкой мыши по **Databases** и выберите **Create → Database**.
5. В поле **Database** укажите имя: `sub_monitor`.
6. Нажмите **Save**.

### Создание таблиц

Выберите созданную базу `sub_monitor`, нажмите **Query Tool** (иконка с изображением молнии) и выполните следующий SQL-скрипт:

```sql
CREATE TABLE users (
    id SERIAL PRIMARY KEY,
    email VARCHAR(255) UNIQUE NOT NULL,
    password VARCHAR(255) NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE subscriptions (
    id SERIAL PRIMARY KEY,
    user_email VARCHAR(255) REFERENCES users(email) ON DELETE CASCADE,
    name VARCHAR(255) NOT NULL,
    price DECIMAL(10,2) NOT NULL,
    category VARCHAR(100),
    period INTEGER,
    period_unit VARCHAR(10) DEFAULT 'months',
    last_billing DATE,
    source VARCHAR(255),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

Затем нажмите **Execute** (▶️). Таблицы будут созданы.

## 🚀 Запуск проекта (пошаговая инструкция)

### 1. Клонирование репозитория
```bash
git clone https://github.com/BogdanAn18/tech_nn.git
cd tech_nn
```

### 2. Запуск серверной части (Backend)

```bash
cd backend
npm install
```

Создайте в папке `backend` файл `.env` со следующим содержимым:

```env
JWT_SECRET=mySuperSecretKey
DB_PASSWORD=пароль_от_postgres
```

Если вы изменяли параметры подключения к базе, укажите их явно:

```env
DB_HOST=localhost
DB_PORT=5432
DB_USER=postgres
DB_NAME=sub_monitor
```

Запустите сервер:

```bash
node server.js
```

При успешном запуске вы увидите в консоли:

```
✅ База данных подключена
🚀 Сервер на http://localhost:5000
```

### 3. Запуск веб-интерфейса (Frontend)

Откройте **новый терминал**, не закрывая предыдущий.

```bash
cd frontend
npm install
npm run dev
```

Перейдите в браузере по адресу `http://localhost:3000`. Должна открыться главная страница приложения.

### 4. Запуск мобильного приложения

Ещё один терминал:

```bash
cd device_new
npm install
npm start
```

После запуска Metro Bundler в терминале появится QR-код.  
Установите на своём мобильном устройстве приложение **Expo Go** (доступно в App Store и Google Play) и отсканируйте им QR-код. Мобильное приложение запустится.

## 🔗 Взаимодействие компонентов

- **Сервер** работает по адресу `http://localhost:5000` и ожидает запросы от клиентов.
- **Веб-клиент** (порт `3000`) и **мобильное приложение** отправляют запросы на этот адрес.
- Если вы тестируете мобильное приложение на реальном устройстве, компьютер и телефон должны находиться в одной сети Wi‑Fi.  
  В файле `src/api/client.js` мобильного проекта необходимо заменить `localhost` на **локальный IP-адрес вашего компьютера** (узнать его можно командой `ipconfig` в Windows или `ifconfig` в macOS/Linux).  
  Пример: `http://192.168.1.10:5000`.

## ❗ Возможные трудности и способы их решения

| Проблема | Решение |
|----------|---------|
| `Error: password authentication failed` | Проверьте, что пароль в файле `.env` совпадает с паролем, заданным при установке PostgreSQL. |
| `Error: database "sub_monitor" does not exist` | Убедитесь, что база данных `sub_monitor` создана в pgAdmin. |
| `Error: role "postgres" does not exist` | При установке PostgreSQL вы могли указать другое имя пользователя. Пропишите его в `.env` (например, `DB_USER=myuser`). |
| Мобильное приложение не подключается к серверу | Проверьте, что в коде мобильного приложения указан правильный IP-адрес компьютера. Убедитесь, что брандмауэр Windows разрешает входящие подключения для Node.js. |
| При загрузке на GitHub папки отображаются как «стрелки» (подмодули) | Внутри папок `frontend` или `device_new` остались скрытые каталоги `.git`. Удалите их командой `rm -rf frontend/.git` (и аналогично для других) и сделайте новый коммит. |
